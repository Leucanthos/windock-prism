1.1 窗口枚举 i

实现

DockManager.FullRefresh() — 启动/Toggle 时触发。遍历 GetTopWindow → GetWindow(GW_HWNDNEXT) 枚举所有顶层窗口，按 PID 去重，每个 PID 取最长标题的窗口。

每 1.5s badgeTimer 更新角标计数，不重建图标。此前 DockBar.RefreshIcons() 的定时器因 DockBar.Create() 从未被调用而完全未启动。

 !

踩坑

DockBar 死代码：DockBar.Create() 从未被调用，refreshTimer 从未启动。Badge 只在 FullRefresh 时更新一次，用户打开/关闭窗口后不刷新。修复：在 DockManager.Create() 加独立 badgeTimer。

Debug 模式位置陷阱：Debug 模式下 Dock 在 Y=40（顶部），导致图标检测启发式（仅检查底部）失败。

1.2 Badge 计数规则 i

实现

CountWindows(pid) — 遍历所有窗口，匹配 PID + IsVisibleWindow。

CountWindowsForPin(pinPath) — 遍历所有可见窗口，对每个匹配 pin 的 PID 调用 CountWindows 求和。解决多进程浏览器（Edge 8+ 进程）窗口分散问题。

badgeTimer 每 1.5s 调用上述方法更新 badge，不重建图标。

 !

踩坑

seenPids 检查顺序错误：CountWindowsForPin 初版在 IsWindowVisible 之前检查 seenPids.Add(pp)，导致第一个不可见窗口就把 PID 标记为"已处理"，后续同 PID 的可见窗口全被跳过。

PID 死亡 Badge 归零：关闭窗口时特定 PID 进程退出，GetProcessById 抛异常 → catch { } 吞掉 → badge=0。修复：改用 di.PinPath 计数，不依赖 di.Pid。

微信 8 个子窗口：CountWindows 统计所有可见窗口，微信内部子窗口被误计。修复：WeChat/Steam 托盘应用特殊处理，始终显示 1。

踩坑

双重刷新路径：FullRefresh()（启动/Toggle）和 RefreshIcons()（每 3s 定时）是两条独立路径。FullRefresh 的 pinned 去重修好了，但 RefreshIcons 仍在创建重复图标。修复后在 RefreshIcons 也加了 pinned 进程跳过 + 图标保留逻辑。

2.3 后台进程检测 + 点击恢复 i

实现

后台进程检测：可见窗口枚举匹配失败 → GetProcessesByName(pinName) 检查进程是否后台运行。对所有匹配进程比较 CountWindows(pid)，选可见窗口最多的 PID。

通用点击恢复：di.HWnd 有效 → FocusWindow（IsIconic→SW_RESTORE, !IsVisible→SW_SHOW → SwitchToThisWindow）。无效 → Process.Start(path)。

微信 X-close 恢复（最终方案）：微信 X 关闭后进入 vis=False, iconic=False, 894×647 死快照状态。任何 Win32 消息均无法恢复交互性。唯一有效方法：ShowWindow 显示快照 → SendInput（原子 MOVE+LEFTDOWN+ABSOLUTE）硬件点击最小化按钮 (Right-188, Top+18) → Qt5 退出快照模式 → SW_RESTORE 恢复活窗口。

最小化关闭：iconic=True, 158×26 → SW_RESTORE 直接恢复（无需 SendInput）。

 !

踩坑

约 20 种失败方案：SW_SHOW/RESTORE/SHOWNORMAL、SetForegroundWindow、SwitchToThisWindow、AttachThreadInput、WM_ACTIVATE 脉冲、WM_SYSCOMMAND(SC_MINIMIZE)、RegisterWindowMessage 0xC000-0xFFFF 全枚举、WH_GETMESSAGE 钩子(Error 183)、WM_USER+NIN_SELECT 枚举(0-101)、keybd_event Alt+Tab 等均无效。

SendInput 原子性：MOVE + LEFTDOWN 需合并在一个 INPUT 事件中，否则用户鼠标移动会在 MOVE 和 DOWN 之间插入干扰。

MainWindowHandle 为 0：ApplyForegroundToIcon 要求 MainWindowHandle != IntPtr.Zero 才设置 Pid。Steam 启动慢，300ms 触发时 Handle 还是 0 → 跳过。修复：只要进程存在就设置 Pid。

FindWindow ANSI 限制：DockManager 的 FindWindow 默认是 ANSI 版本，无法匹配中文窗口标题"微信"。需加 CharSet=CharSet.Unicode。

检测流程
枚举可见窗口→ 按路径/进程名 匹配 PinnedPaths→ ✓ 匹配 → pinned+running
✗ 未匹配→ GetProcessesByName → 选窗口最多的 PID→ 进程存活 → 后台 pinned+running→ 不存在 → pinned-inactive

4.3a工作区释放 i

实现

ABM_REMOVE：SHAppBarMessage(ABM_REMOVE, tb) 取消任务栏的 appbar 注册，Windows 释放预留的 48px 工作区。此前 SPI_SETWORKAREA 尝试失败——系统会立即还原。

恢复：ABM_NEW + ABE_BOTTOM 重新注册任务栏为底部 appbar，再 ShowWindow。

守护：themePoll 每 400ms 检查任务栏是否可见，可见则 ShowWindow(tb, 0)。

 !

踩坑

SPI_SETWORKAREA 无效：设置后工作区确实变成全屏，但系统（Explorer）会立即重新计算并还原。这是 Windows 的预期行为——只有注册的 appbar 才能稳定修改工作区。

ABS_AUTOHIDE 的副作用：ABM_SETSTATE(ABS_AUTOHIDE) 能释放工作区，但同时启用任务栏自动弹出——鼠标触底时 Explorer 会 ShowWindow 任务栏，虽可在 400ms 内藏回但有闪烁。

ABM_REMOVE 后任务栏仍可弹出：Explorer.exe 内部管理任务栏窗口，不依赖 appbar 注册状态。鼠标触底时 Explorer 仍然 ShowWindow 任务栏。

 		✓
4.3bEdge Guard i

实现

透明守护条：全宽 4px TopMost 透明 Form，贴在屏幕底部（Y = Bounds.Height - 4）。鼠标进入底部区域时落在守护条上，Explorer 检测不到屏幕边缘，不会触发任务栏弹出。

BackColor=Black + TransparencyKey=Black 实现完全透明，ShowInTaskbar=false 不在任务栏显示。

关闭 dock 时 edgeGuard.Close() 销毁。

 !

踩坑

DockBar 死代码陷阱：DockBar.Create() 从未被调用，导致分析时误以为 shelf Form 可以遮挡底部。实际运行的是 DockManager.Create()，它只有一个 invisible anchor form，底部完全没有遮挡。

Off-screen SetWindowPos 不防弹出：把任务栏窗口移到屏幕外（Y+100）后，Explorer 在鼠标触底时仍然将其 ShowWindow 回原位。Explorer 的鼠标边缘检测是基于屏幕坐标的，不依赖窗口位置。

4.8最大化自动隐藏 i

实现

Opacity 控制：themePoll 每 400ms 枚举所有可见非自有窗口，检查 IsZoomed()。发现最大化窗口时设所有图标 Opacity=0.0，无最大化时恢复 0.82。用 Opacity 而非 Show/Hide —— 不需要重建布局。

排除自有进程（ownPid），dock 自身的图标不会误触发隐藏。

与 Edge Guard（4.3b）协同：最大化窗口本身遮挡屏幕底部，Edge Guard 是否 TopMost 不影响效果。

 !

踩坑

TopMost 方案失败：设 TopMost=false 后 dock 仍在前面——非 topmost 窗口按激活 Z 序排列，用户点击最大化的窗口不会把 dock 压下去。

HWND_BOTTOM + WS_EX_TRANSPARENT 方案也不够：鼠标穿透了但 dock 依然可见（在 Z 序顶部），因为新建图标通过 Show() 会自动取得焦点。

检查位置关键：auto-layer 必须放在 themePoll 最末尾——FgAdd/StaleRemove/LayoutWithLock 都会在同一个 400ms 周期内创建新图标（DockIcon 构造函数设 Opacity=0.82），先检查的话这些新图标不会被隐藏。

仅检查前景不够：dock 图标在最上层时 GetForegroundWindow() 返回 dock 自己的 Form → IsZoomed 返回 false → 永远不隐藏。需枚举所有可见窗口。
